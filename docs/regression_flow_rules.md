# 真实交易流程回测规则系统
## A股量化系统 — 严格时间线驱动 + 多层级穿透 + 反未来数据

---

## 一、核心原则

```
┌─────────────────────────────────────────────────────────┐
│           三大铁律（违反则回测结论不可信）                │
├─────────────────────────────────────────────────────────┤
│ ① 时间线绝对严格 — 任何选股/买入决策只用T-1(及之前)数据 │
│ ② 买入执行在 T 日 09:25~14:57 — 用腾讯开盘后实时行情    │
│ ③ 卖出执行在 T+N 日 — T+1开盘/收盘/盘中止损            │
└─────────────────────────────────────────────────────────┘
```

**每一笔回测从产生到结束的时间线：**

```
T-1 收盘(15:00)  ← 选股条件全部从这里算
T-1 收盘后(17:30) ← 龙虎榜数据到达
    ↓
T 日 09:15-09:25  ← 集合竞价(只看不操作)
T 日 09:25-09:30  ← 开盘价确定 + 量比确认
T 日 09:30-14:57  ← 买入信号触发 + 盘中执行
T 日 15:00        ← 收盘
    ↓
T+1 日 09:25      ← 卖出决策(开盘卖/竞价卖)
T+1 日 09:30-14:57 ← 盘中止损/止盈
T+1 日 15:00      ← 最终盈亏确定
```

### 2.2 各数据源「选股可用时间点」（回测最严格定义）

| 数据源 | 数据内容 | 回测中选股可用时间点 |
|:-------|:---------|:-------------------:|
| **kline_cache.kline** | 日K线(开/高/低/收/量) | **T-1 收盘15:00即可用** |
| **lhb_list** | 龙虎榜上榜(日/3日) | **T+0 收盘后17:30** |
| **lhb_detail** | 龙虎榜买卖前五+席位 | **T+0 收盘后17:30** |
| **lhb_detail_tdx** | TDX打板类型(sblx/yzmc) | **T+0 收盘后17:00** |
| **em_lhb_cache** | 东财营业部回报(r1~d10) | **T+0 16:30** |
| **market_daily.day_full** | 全市场74字段扫描 | **T+0 15:05** |
| **腾讯实时行情** | 盘中数据 | **实时(仅用于T日买入)** |
| **env_daily_history** | 环境历史(570天) | **T-1即可用** |
| **竞价数据(auction)** | 竞价快照 | **T日09:15~09:25** |

### 2.3 ⚠️ 常见反未来数据陷阱（必须避免）

| ❌ 错误做法 | ✅ 正确做法 | 影响程度 |
|:-----------|:----------|:--------:|
| 用T日龙虎榜数据选T日买入标的 | 只能用T-1及之前的龙虎榜数据选股 | **严重 — 数据不可用** |
| 用T日涨停明细选T日买入标的 | 涨停明细是T日收盘后才有的 | **严重** |
| 用T日市场情绪做T日开盘决策 | T-1市场情绪(从market_daily取) | **严重** |
| 用T+1 K线算T日选股条件 | T日选股只能用T-1及之前K线 | **致命** |
| 回测时不考虑成交量缺失(缩量/跌停无法买入) | 缩量=买不到, 用昨收价+限价模拟 | **中等** |

## 三、交易计划 — 买入卖出时间条件

### 3.1 回测信号类型矩阵

每一个信号来源决定「什么时候能买」

```
信号来源                数据产生时间       买入时间窗       卖出时间窗
─────────────────────────────────────────────────────────────────────
策略A: T-1龙虎榜        T-1 17:30~18:00   T 09:25~14:57   T+1 09:25~15:00
策略B: T-1市场扫描      T-1 15:05~15:10   T 09:25~14:57   T+1 09:25~15:00
策略C: T日竞价数据       T 09:15~09:25     T 09:25~09:30   T+1 09:25~15:00
策略D: T日盘中信号       T 09:30~14:57     T 即时~14:57    T+1 09:25~15:00
策略E: T-1算T+1高开预判  T-1 15:00         T 09:25~09:30   T+1 09:25~15:00
策略F: T-1龙虎榜+3日榜   T-1 17:30         T 09:25~14:57   T+1 09:25~15:00
```

### 3.2 买入执行 — 3种典型场景

**场景1：开盘买（09:25~09:30）**
```
条件：信号在T-1或T-1之前产生
价格：开盘价(open) 或 开盘价的+1%限价(确保成交)
适用：所有T-1龙虎榜选股、T-1环境信号
```

**场景2：盘中买（09:30~14:57）**
```
条件：信号在T日盘中产生(竞价信号/盘中放量)
价格：当前实时价格(取Tick级近似 ← 无法精确，用5分钟聚合)
适用：T日竞价确认、T日量比确认、T日盘中放量脉冲
```

**场景3：集合竞价确认买（09:25 基于竞价结果）**
```
条件：T-1已选出候选池 + T日竞价数据确认
价格：开盘价(open)
适用：竞价高开>0%且量比<0.7的缩量板确认
```

### 3.3 卖出执行 — 4种卖出规则

| 卖出规则 | 执行窗口 | 价格定义 | 适用条件 |
|:---------|:--------|:--------|:---------|
| **T+1开盘卖** | T+1 09:30 | 开盘价(open) | 回测研究"开盘卖vs收盘卖"差异 |
| **T+1收盘卖** | T+1 15:00 | 收盘价(close) | 默认规则:持股1天 |
| **T+1止损卖** | T+1 09:30~14:57 | 首次触发-5%的价格 | 开盘低于买入价-5%则立即卖 |
| **T+1竞价低开卖** | T+1 09:25~09:30 | 开盘价 | 低开>-3%竞价时段卖 |

**卖出决策流程：**
```
T+1 09:15 ── 竞价数据可用
     │
     ├─ 竞价低开 > -3% → 开盘卖(卖出策略A)
     │
     ├─ 竞价涨 > +3%  → 观望到收盘(卖出策略B)
     │
     ├─ 竞价 -3%~+3%  → 等待盘中信号
     │
     └─ 若无T+1数据 → 默认收盘卖
```

### 3.4 价格映射规则（回测价格定义）

| 场景 | 买入价 | 卖出价 | 滑点假设 |
|:-----|:------|:------|:--------|
| 开盘买/卖 | `open`(开盘价) | `open`(开盘价) | 0.2%~0.5% |
| 收盘买/卖 | `close`(收盘价) | `close`(收盘价) | 0.1% |
| 盘中买 | 取(open + high) / 2 | — | 0.5% |
| 盘中卖(止损) | — | 止损触发价的-1% | 1.0% |
| 涨停板买入 | 涨停价(昨收×1.1) | — | 可能无法买到 |
| 跌停板卖出 | — | 跌停价 | 可能无法卖出 |

## 四、监控买入信号和执行流程

### 4.1 买入信号生成时间轴（每日）

```
08:30 ── 早盘第一次全链路跑分(morning_pipeline)
         │ 数据源: kline_cache(昨收) + market_daily(前天) + lhb_cache(前天盘后)
         │ 输出: 环境评分 + 策略匹配 + S级候选池
         │ ❌ 无龙虎榜数据 → 使用前一日市场环境
         ↓
09:15 ── 集合竞价开始
         │ 采集竞价快照(腾讯行情)
         │ 采集东财竞价API(3次快照)
         ↓
09:25 ── 开盘价确定 + 量比确认
         │ 买入信号触发条件:
         │ ① 候选池中量比<0.7(缩量确认)
         │ ② 竞价高开>0%(趋势延续)
         │ ③ 竞价成交额>前5日均值/10(有量能)
         ↓
09:30~14:57 ── 盘中轮询(每5分钟)
         │ 腾讯实时行情检查:
         │ ① 当前价 < 开盘价+2%(未大幅拉升)
         │ ② 当前量比 < 0.7(持续缩量)
         │ ③ 未封死涨停(还能买入)
         ↓
盘中信号触发提示 → 人工/自动确认 → 执行买入
```

### 4.2 买入条件综合评分（T日09:25后）

```
每个候选股计算「可买性评分」= f(宏观环境, 个股资金, 竞价信号, 技术面)

score = env_score × 0.2
      + fund_score × 0.3    (机构/游资/北向)
      + auction_score × 0.25 (竞价涨幅+量比+成交额)
      + tech_score × 0.15    (MA5/趋势/量价)
      + limit_score × 0.10   (封板强度/板数)

阈值:
  ≥ 75分: 强力买入(S级)
  ≥ 60分: 正常买入(A级)
  ≥ 40分: 谨慎买入(B级)
  < 40分: 放弃
```

### 4.3 买入候选池来源

| 来源 | 数据时间 | 包含数量 | 可信度 |
|:-----|:--------|:--------:|:------:|
| lhb_list(龙虎榜日榜type=01) | T-1盘后 | ~20~50只 | ⭐⭐⭐⭐⭐ |
| lhb_list(3日榜type=04) | T-1盘后 | ~10~30只 | ⭐⭐⭐⭐☆ |
| K线涨停扫描(3日内涨停过) | T-1收盘 | ~50~200只 | ⭐⭐⭐☆☆ |
| 概念热点 | T-1收盘~T日开盘 | 不定 | ⭐⭐☆☆☆ |
| 竞价异常检测 | T日竞价 | ~5~15只 | ⭐⭐⭐⭐☆ |

## 五、监控卖出信号和执行流程

### 5.1 卖出信号触发机制

```
T日买入 → T+1日开盘前卖出决策
                │
         ┌──────┴──────┐
         │               │
    T+1竞价数据     T+1历史回测
         │               │
    竞价低开>3%?    策略卖出建议
    竞价高开>5%?     
         │               │
         └──────┬──────┘
                │
           综合卖出决策
                │
    ┌───────┬───┴───┬───────┐
    │       │       │       │
  竞价卖  开盘卖  盘中止损  收盘卖
```

### 5.2 卖出规则优先级

```
优先级1: 止损（无条件执行）
  └─ T+1盘中价格 ≤ 买入价 × 0.95 → 立即卖出

优先级2: 竞价低开卖出
  └─ T+1竞价低开 > 买入价 × 0.97 → 开盘集合竞价卖

优先级3: 竞价高开持有
  └─ T+1竞价高开 > 买入价 × 1.03 → 持有到收盘

优先级4: 默认收盘卖出
  └─ 未触发任何卖出条件的，T+1收盘卖
```

### 5.3 卖出确认单（格式）

```
┌─ 卖出执行确认单 ─────────────────────┐
│ 🔔 SELL SIGNAL: T+1 09:27             │
│ ────────────────────────────────────│
│ 🎯 标的: 603986.SH 兆易创新           │
│ ├ T日买入价: 137.28                    │
│ ├ T+1竞价: 141.80 (+3.29%)            │
│ └ 持仓盈亏: +3.29% ✓                  │
│ ────────────────────────────────────│
│ 📋 卖出策略:                           │
│ 竞价高开>+3% → 持有到收盘 (策略B)     │
│ ────────────────────────────────────│
│ ✅ 无卖出行 — 继续持有，等待收盘信号   │
└─────────────────────────────────────────┘
```

## 六、回测执行 — 完全时间线驱动的批量回测

### 6.1 回测周期定义

每一笔交易的完整周期：

```
T-1 (选股日) → T (买入日) → T+1 (卖出日)
  │              │              │
  选股条件       买入执行       卖出计算
  只用T-1        T 09:25        T+1 收盘价
  及之前数据     开盘价买        或开盘价
```

**关键约束：**
- `T日` = 买入执行日（有kline数据，09:30~15:00）
- `T+1日` = 卖出日（有kline数据的下一个交易日）
- 如果T+1是休市日 → 顺延到下一个交易日
- 如果T+1无kline数据 → 跳过该笔（非交易日）

### 6.2 回测SQL查询模版（反未来数据验证核心）

```sql
-- 核心：T-1龙虎榜选股 → T日开盘买入 → T+1收盘卖出
-- 只使用 T-1 及之前的数据做选股条件
SELECT 
  l.date AS t_minus_1_date,
  l.code AS stock_code,
  l.name AS stock_name,
  l.chg AS t_minus_1_chg_pct,
  
  -- T日开盘价(买入价)
  k_t.open AS buy_price,
  
  -- T+1日收盘价(卖出价)
  k_t1.close AS sell_price,
  
  -- 收益率
  (k_t1.close - k_t.open) / k_t.open * 100 AS t1_return
  
FROM lhb_list l

-- T日K线(用于取开盘价 = 买入价)
JOIN kline k_t 
  ON l.code = k_t.code 
  AND k_t.date = DATE(l.date, '+1 day')

-- T+1日K线(用于取收盘价 = 卖出价)
JOIN kline k_t1 
  ON l.code = k_t1.code 
  AND k_t1.date = DATE(l.date, '+2 day')

WHERE l.type = '01'  -- 仅日涨幅偏离值≥7%(打板标的)
  AND k_t.open IS NOT NULL
  AND k_t1.close IS NOT NULL
```

### 6.3 回测五大维度（每维度可独立开关）

```
维度A: 资金维度
  └─ 龙虎榜买方席位特征
  └─ 机构/游资/量化出现次数
  └─ 北向资金参与与否

维度B: 个股维度  
  └─ T-1日换手率/量比
  └─ T-1日涨停是否一字板
  └─ T-1日封板时间(早/中/尾)

维度C: 环境维度
  └─ T-1日涨停数/炸板率
  └─ T-1日市场情绪(恐慌/活跃/高潮)
  └─ T-1日大盘涨跌幅
  └─ T-1日涨跌比

维度D: 时间维度
  └─ 星期几(周内效应)
  └─ 月份(季节效应)
  └─ 季度末效应
  └─ T-1距离上次涨停天数

维度E: 组合维度(多条件叠加)
  └─ 资金×环境×个股
  └─ 信号叠加:N条件→收益递进
```

## 八、多层级穿透式回测规则系统架构

### 8.1 穿透层级定义

```
L6 宏观周期层 ──────────────── 市场处于什么大环境
  R6-1: 全市场滚动胜率周期
  R6-2: 季节效应(月份)

  R5-1: 北向资金强度
  R5-2: 游资活跃度

  R5-4: 散户指标(RTI)
  R5-5: 机构参与度

L4 环境匹配层 ──────────────── 什么环境下用什么策略
  R4-1: 冰点策略(涨停<17)
  R4-2: 震荡策略(涨停17~40)
  R4-3: 活跃策略(涨停40~70)  
  R4-4: 高潮策略(涨停>70)
  R4-5: 切换检测

L3 个股筛选层 ──────────────── 什么样的个股值得买
  R3-1: 首上榜(t-1首次出现)
  R3-2: 净买额(买入>卖出)
  R3-3: 量能×资金类型(缩量游资最优)
  R3-4: 技术形态(MA5/趋势)

  R2-1: 赚钱席位(>2%,>20笔)
  R2-2: 亏钱席位(回避)
  R2-3: 机构+游资双买
  R2-4: 游资+量化组合
  R2-5: TDX标签(炒股养家/量化打板/T王)

L1 操作执行层 ──────────────── 最终买/卖决定
  R1-1: 仓位分配(甲等50%/乙等30%/丙等10%)
  R1-2: 买入模式(主升追涨/回踩低吸/缩量板)
  R1-3: 风控规则(止损-5%/仓位上限)
  R1-4: 卖出策略(T+1开盘/收盘/止损)
  R1-5: 确认清单(10项必查)
```

### 8.2 穿透计算规则

```
每层输出 = Σ(规则节点_i × 置信度_i × 权重_i) / Σ(置信度_i × 权重_i)

最终决策 = L6 × 0.10 + L5 × 0.15 + L4 × 0.15 + L3 × 0.20 + L2 × 0.20 + L1 × 0.20

置信度动态调整(基于T+1红黑榜反馈):
  命中(+) → 置信度+5%
  未命中(-) → 置信度-5%(不低于15%)
  连续命中5次 → 权重+10%
  连续错过3次 → 权重-20%
```
