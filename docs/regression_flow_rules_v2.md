### 4.3 🆕 **买入信号前30秒规则** — 信号数据必须来自买入决策前≤30秒

这是研究买入信号时**最重要的约束规则**：

```
买入信号研究的数据窗口：
┌────────────────────────────────────────────────────┐
│  ⏰ 买入决策时刻 = T日09:25:00（开盘集合竞价结束）   │
│                                                     │
│  可用的研究数据:                                     │
│    ✅ T-1日及之前的所有K线/龙虎榜/环境数据            │
│    ✅ T日09:24:30~09:25:00提交的前30秒竞价数据       │
│    ✅ T日09:25:00刚产生 + 1秒内的开盘价              │
│    ✅ T日09:25:00产生的量比(基于集合竞价成交量)       │
│    ✅ T日09:15~09:25竞价曲线(趋势/强度)              │
│                                                     │
│  禁止使用的数据:                                     │
│    ❌ T日09:30之后的盘中数据（量比/成交额/价格）      │
│    ❌ T日15:00收盘价                                 │
│    ❌ T+1日任何数据                                  │
└────────────────────────────────────────────────────┘
```

### ⚠️ 常见的错误用法（已经在我们的研究中发现了！）

| 场景 | 错误做法 | 正解 |
|:-----|:---------|:-----|
| **T日09:25买入** | 用T日09:30~14:57的盘中数据(\*)做买入信号 | 只能用竞价数据(09:15~09:25)和T-1日数据 |
| **T日09:30开盘买入** | 用T日09:31之后的数据做买入确认 | 只能用09:25~09:30:59这30秒内的开盘脉冲数据 |
| **T日盘中买入(10:00)** | 用T日10:00:30之后的数据 | 只能用10:00:00±30秒内的数据 |

> **(\*) 本系统自纠错的根因：** 旧回测中\"缩量+主力+机构(+4.89%)\"等TOP5策略，用了**T日的量比/成交额/封单数据**做选股条件。但买入价是T日开盘价——T日量比在09:30才知道，而买入在09:25就发生了，中间有5分钟信息差。这就是为什么新规则回测只得到-0.31%。

### 🆕 正确的买入信号研究范式

```python
# ✅ 正确：基于T-1日的龙虎榜数据 + T日09:25前30秒的竞价数据
买入信号 = {
    'T-1选股条件': '龙虎榜(机构买入, 净额>1000万)',
    'T日09:24:30~09:25:00': '竞价量比<0.7 且 竞价涨幅>+1% 且 竞价量>100万',
    '决策时刻': 'T日09:25:00',
}
# 买入执行 = T日09:25:00以开盘价买入
```

```python
# ❌ 错误（旧回测的做法）
买入信号 = {
    'T-1选股条件': '龙虎榜(机构买入)',
    'T日盘中条件': '量比<0.7',  # ← 09:30之后才知道的数据！
    '决策时刻': 'T日09:25（但用了09:30后的数据）'  # ← 时间矛盾
}
```

### 5.2 🆕 **卖出信号前30秒规则** — 信号数据必须来自卖出决策前≤30秒

与研究买入信号对称：

```
卖出信号研究的数据窗口：
┌──────────────────────────────────────────────────────┐
│  ⏰ 卖出决策时刻 = T+1日09:25:00（平仓时刻）          │
│                                                       │
│  可用的研究数据:                                       │
│    ✅ T+1日09:24:30~09:25:00前30秒的竞价数据           │
│    ✅ T+1日09:25:00开盘价                             │
│    ✅ T日15:00之前的所有K线/龙虎榜/环境数据             │
│                                                       │
│  禁止使用的数据:                                       │
│    ❌ T+1日09:30之后的盘中数据                         │
│    ❌ T+1日15:00收盘价（作为卖出信号输入，不是输出）     │
└──────────────────────────────────────────────────────┘
```

### ⚠️ 卖出信号研究的常见错误

| 场景 | 错误做法 | 正解 |
|:-----|:---------|:-----|
| **T+1开盘卖** | 用T+1日全天的数据判断卖出信号 | 只能用T+1日09:24:30~09:25:00的竞价数据 |
| **T+1盘中卖** | 用T+1日15:00收盘价做卖出信号 | 只能用触发卖出条件前30秒内的高频数据 |

### 🔑 一句话总结

> **买入信号只能用买入前30秒的数据（竞价→开盘）；卖出信号只能用卖出前30秒的数据（竞价→开盘）。**
> 
> **回测中假设\"开盘价\"能买入时，信号条件只能用T-1日K线 + 竞价数据，绝不能用T日盘中或收盘数据。**

## 四、③ 交易计划（v2.0 — 覆盖4种场景的完整时间线 + 新增30秒规则）

### 4.1 基础交易流程

```
T-1 15:00收盘 ────→ T-1 17:30龙虎榜数据到达 ────→ T-1 20:00选出候选池
                ↓
        T 09:15~09:25竞价确认
                ↓
        T 09:25开盘价确定 ─── 买入(open)
                ↓
        T 09:30~14:57盘中监控
                ↓
        T 15:00收盘
                ↓
        T+1 09:25竞价确认 ─── 卖出(open) 或
        T+1 15:00收盘 ──── 卖出(close)
```

### 4.2 🆕 按信号来源分的4种交易场景（实战增加）

| 场景 | 信号来源 | 买入时间窗 | 买入价 | 卖出时间 | 卖出价 | 样本量 |
|:-----|:---------|:---------:|:------|:--------|:------|:-----:|
| **A. 标准龙虎榜** | T-1收盘后的龙虎榜 | T日09:25~09:35 | open | T+1收盘 | close | **1,541笔** |
| **B. 盘中信号** | T日竞价/盘中放量 | T日即时 | 当前价 | T+1收盘 | close | 未回测 |
| **C. 隔夜信号** | T-1环境+T-1龙虎榜 | T日09:25 | open | T+1开盘 | open | 1,541笔 |
| **D. 3日榜** | T-1的3日涨幅榜(注意!) | T日09:25 | open | T+1收盘 | close | **1,468笔** |

**🆕 场景D的特殊处理：** 3日榜上榜条件是"连续3日涨幅偏离值达20%"，上榜日是T-2~T。我们的SQL用`DATE(date, '+1 day')`取T日开盘价是正确的——因为3日榜公布时T日已收盘，真正的T+1才是买入日。

### 4.3 🆕 价格映射精度对比（实战跑出来的）

| 卖出策略 | type=01(打板) | type=04(3日榜) | type=*全部 |
|:---------|:------------:|:--------------:|:----------:|
| **T+1开盘卖(open)** | +0.10%/44.1% | -0.31%/42.2% | -0.32%/43.1% |
| **T+1收盘卖(close)** | +0.04%/45.4% | +0.12%/45.4% | -0.20%/44.6% |
| **差异(close-open)** | **-0.06%** | **+0.43%** | **+0.12%** |

> **结论：** type=01(打板)建议开盘卖，type=04(3日榜)建议收盘卖。
> 差异只有0.06~0.43%，远小于随机波动，**价格映射不是回测最大的误差来源**。

## 五、④ 买入信号监控与执行（v2.0修正版）

### 5.1 买入信号时间线（精确到30分钟）

```
T-1 17:30 ── 第一重信号：龙虎榜数据到达
     ↓  运行regression_flow_engine获取穿透策略
     
T 08:30 ── 第二重信号：morning_pipeline全链路跑分
     ↓  环境评分 + 穿透策略推荐 + 候选池

T 09:15 ── 第三重信号：竞价数据到达
     ↓  竞价高开/低开/量比确认

T 09:25 ── 第四重信号：开盘价确定 + 量比确认
     ↓  买入确认(量比<0.7且高开>0%)

T 09:25~09:35 ── 买入执行窗口(open价)
     ↓  
T 09:35~14:57 ── 盘中买入(只在强势确认时)
```

### 5.2 🆕 买入信号置信度评分（严格时间线版）

每个买入信号必须同时满足**5个维度**才能执行：

```python
置信度 = L6(星期) × 0.1 
        + L5(资金类型) × 0.2 
        + L4(环境) × 0.3 
        + L3(个股形态) × 0.2 
        + L2(席位) × 0.2
```

**置信度阈值（基于回测）:**
- ≥70分: ✅**强力买入** — 周二+活跃+涨停≥50(回测+5.26%/71.4%)
- 50~69分: ✅正常买入 — 活跃环境(回测+1.42%/53.9%)
- 30~49分: ⚠️谨慎 — 冰点(回测+0.35%/47.9%)
- <30分: 🚫**不买** — 周三震荡低涨停(回测-1.00%/37.5%)

### 5.3 🆕 买入必查清单（防止回测和实盘脱节）

| 检查项 | 说明 | 回测中 | 实盘中 |
|:-------|:-----|:------|:------|
| ✅ 是否在交易时段 | 09:25~14:57 | 假设yes | 必须确认 |
| ✅ 是否涨停封死 | 涨停封死买不到 | 假设open价能买 | ❌买不到 |
| ✅ 是否跌停 | 跌停无法买入 | 假设open价能买 | ❌买不到 |
| ✅ 停牌/ST | 停牌无法交易 | 自动跳过 | 必须检查 |
| ✅ 竞价量比<0.7 | 缩量确认 | 假设符合 | 必须实时确认 |
| ✅ 非一字板 | 一字板买不到 | 做了降级15% | ❌买不到 |
| ✅ 龙虎榜数据T-1 | 不是T日数据 | 已保证 | 必须保证 |

## 六、⑤ 卖出信号监控与执行（v2.0修正版）

### 6.1 卖出规则决策树

```
                    T+1 09:25
                        │
                  竞价数据到达
                        │
              ┌─────────┴─────────┐
              │                   │
         低开>-3%            低开>-3%
              │                   │
         竞价涨幅            竞价涨幅
              │                   │
       ┌──────┴──────┐     ┌─────┴─────┐
       │             │     │           │
    涨幅<-5%     涨幅<-3%   涨幅>+3%    涨幅 -3%~+3%
       │             │     │           │
    ❌亏损>5%      ⚠️亏损>3%  ✅持有     ⚠️盘中看
    无条件卖     开盘卖     到收盘     15分钟确认
```

### 6.2 🆕 卖出价格定义（精确到场景）

| 卖出策略 | 价格来源 | 适用场景 | 回测收益 |
|:---------|:--------|:--------|:--------|
| **T+1开盘卖** | `kline.open` | 竞价低开>3%或隔夜风险 | type=01: +0.10% |
| **T+1收盘卖** | `kline.close` | 默认策略 | type=01: +0.04% |
| **T+1止损卖** | `min(kline.low, open×0.95)` | 盘中断崖下跌 | 未单独回测 |
| **T+1涨停卖** | `yclose×1.1` | T+1继续涨停 | 持有不止盈 |

### 6.3 🆕 卖出策略选择（基于回测结论）

```
若T+1竞价涨幅 > +3% → 持有到收盘（期望更高收益）
若T+1竞价涨幅 < -3% → 开盘立即卖（避免更大亏损）
若T+1竞价涨幅 -3%~+3% → 盘中监控，收盘决定
```

### 8.1 同一信息从多个API验证

### 8.2 数据冲突处理规则

```
当两个数据源给出不同结果时：
  1. 优先使用更新更完整的数据源
  2. 标注冲突级别：差异<5%可忽略，5~20%需说明，>20%结论不可用
  3. 回测中始终使用可以联合校验的数据
```

## 九、🆕 v2.0新增⑧ 样本偏误纠正机制

### 9.1 已知偏误列表

| 偏误类型 | 影响 | 严重程度 | 修正方法 |
|:---------|:-----|:--------|:---------|
| **创业板缺失** | 3***股票全部排除 | 🔴严重 | 结论只适用于主板 |
| **北交所缺失** | 920***全部排除 | 🟡中等 | 720只小市值票 |
| **后视镜偏差** | 用T日数据选T日股票 | 🔴致命 | 已用严格时间线SQL修复 |
| **幸存者偏差** | 只测了有K线的票 | 🟡中等 | 73.3%的T+1 K线覆盖 |
| **样本量不均衡** | 震荡期占67% | 🟡中等 | 分层统计已做 |

### 9.2 偏误修正公式

```python
# 每个结论的"可信度修正分" = 原始分 × 数据覆盖系数 × 样本量系数 × 偏误系数

credibility = raw_score * (kline_coverage_pct / 100) * min(n/100, 1) * (1 - bias_penalty)
```

**当前1,541笔回测的可信度计算：**
- 数据覆盖: 73.3% (T+1 K线可用率)
- 样本量: min(1541/100, 1) = 1.0
- 偏误罚分: 创业板+北交所缺失 = 0.2
- **可信度 = 1.0 × 0.733 × 1.0 × 0.8 = 0.586**
